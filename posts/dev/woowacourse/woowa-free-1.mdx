## 우아한테크코스 웹 프론트엔드 5기 프리코스 1주차 소감문

- ### **[Community - Test Case]**

  - slack에서 많은 사람들과 이야기 나눌 수 있게 커뮤니티 장을 열어준 것이 너무 마음에 들었습니다. 다른 사람들이 어떤 트러블을 겪었고 트러블에 대한 해결책을 제시해주는 소통의 장이 다른 개발자가 성장하는 발판이 된다고 생각이 들었습니다. 과제를 하는 도중 Test Case를 공유하는 방이 생겼고, 바로 들어가 다른 개발자들이 생각하는 문제의 해석과 제가 미처 생각하지 못했던 예외들이 많았습니다. 예를 들어 Problem 2번에는 '연속하는 중복 문자' 를 연속이 pair라고 생각하기도 하고 연속되기만 하면 다 중복제거를 해야한다라고 생각하는 개발자들도 있었습니다. 이런 소통의 장에서 다른 개발자들의 말에 귀기울이며 더욱 성장하는 계기가 된 것 같습니다.

- ### **[기능 목록 작성]**

  - 문서를 보고 그대로 구현을 할 수 있는 능력도 개발자의 중요한 능력이라고 생각합니다. 저는 여태껏 프로젝트를 하면서 기능별로 적고 구현하기 보다는 대략적인 기능을 상상하고 그 코드를 그대로 타이핑하면서 구현을 해왔습니다. 하지만 이번 과제를 통해 먼저 어떻게 구현 할 것인지 기능 구현 목록을 작성하고 예외처리는 어떻게 할 것인지, 그리고 예제까지 써보았습니다. 이미 작성된 기능 구현 목록을 보며 코드를 써내려가니, 더 빠르고 꼬이지 않는 구현을 할 수 있었던 것 같습니다. 기능 문서 작성능력을 이번 우아한테크코스를 통해 많이 고민하며 성장시켜 어떠한 설계 문서에도 당황하지 않고 구현할 수 있는 개발자가 되기 위해 노력할 것입니다.

- ### **[Git - Commit convention]**

  - commit convention은 개발자들과의 소통에서 중요한 부분이라고 생각했습니다. 태그만 보더라도 where/what이 한눈에 보여야 좋은 commit이라 생각하여 feat/fix/refactor/docs로 구분하고 그 뒤에 [P1] 을 붙여 어떤 문제의 무슨 commit인지 알아볼 수 있게 노력했습니다.

- ### **[Git - rebase]**

  - 깨끗한 commit은 다른 개발자들이 보기에도 읽기 쉽고 clean합니다. commit을 태그별로 진행하다보니 자꾸 commit이 꼬이는 상황이 발생했습니다. git rebase -i HEAD~명령어로 squash와 reword를 사용해보며 local에서 commit을 깨끗하게 만든 뒤 origin branch에 push하는 작업을 많이 하려고 노력했고 이를 통해 다른 사람들과 git으로 협업을 할 때 조금 더 읽기 편한 commit을 작성하는데 한걸음 다가가려고 했습니다.

- ### **[JS - nesting function]**

  - Javascript에서 중첩함수를 써야할지, 함수를 쓰지 않는 코드가 나은지 많이 고민했습니다. 함수네이밍만 잘 설정한다면, 클린코드관점에서도 가독성이 올라가며 중첩함수는 함수내부에만 사용된다는 것을 명시해놓는 것이기 때문에 분명 가독성이 좋다는 것을 깨달았습니다. 처음엔 함수로 빼내는 것이 성능에 차질이 생기지 않을까 걱정했지만, 검색해보니 최근 브라우저와 JS엔진들은 성능적 차이가 거의 없다고 하여 코드 재사용도 용이한 중첩함수를 사용하자고 생각했습니다.

- ### **[정규표현식 & 반복문]**

  - 저의 과제는 반복문의 향연 이였습니다. valid값을 체크하는 것, 문자열을 검색하는 것, 추출하는 것 등 모든 것에 forEach,map이 들어가 코드가 지저분해지고 최적화가 되지 않았습니다. 하지만 이번 과제 중 PROBLEM 6번을 풀면서 한글을 체크하는데 정규표현식을 처음 써보았습니다. 전과 달리 string배열을 탐색, 검사하는 데도 test함수 하나면 끝나는 정규표현식은 저의 코드를 새롭게 바꿀 수 있는 발판을 마련했지만, 생각보다 짧은 시간에 정규표현식을 활용한 리팩토링은 하지 못했습니다. 이번 1주차가 아닌 2주차의 과제는 좀 더 가독성있고 클린한 코드를 짜보자고 다짐했습니다.

- ### **[JS만의 독특한 문법들(map/reduce/arrowfunction...)]**

  - 아직 저의 코드는 javascript라고 보기 힘든 것 같습니다. 다른 js개발자들의 코드를 보면 다양한 js만의 독특한 함수들을 사용해 다른 언어에서는 if, for, while등으로 구성된 몇십줄의 코드를 단 몇줄로 줄이는 가독성있고 누가봐도 깨끗한 코드를 볼 수 있었습니다. 앞으로의 3주차 미션동안 저는 js의 엄청난 기능들을 전부 활용할 것이며 누가 봐도 깨끗한 코드를 짜려고 노력할 것입니다.
