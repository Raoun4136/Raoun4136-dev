## 우아한테크코스 웹 프론트엔드 5기 프리코스 3주차 소감문

- ### **[클래스(객체)를 분리하는 연습]**

  - 객체를 분리하는 것에대해 저는 프로그래밍의 ‘주체’가 되는 것들을 생각했습니다. 로또게임을 한다면 로또를 사는 사람과 로또를 진행하는 곳으로 분류할 수 있다고 생각했고, App.js라는 클래스를 로또를 진행하는 곳, 그리고 User객체로 분리하여 구현했습니다.
    주체가 동작하는 것을 생각해보며, User는 Lotto객체를 여러개 들고 있을 수 있고 Lotto에 대한 행동들을 할 수 있다고 생각해서 lotto가 담겨있는 배열과 buyLotto, checkLotto등 로또에 관한 로직들을 담당하게 했습니다.
    로또 숫자를 각각 당첨번호와 비교하는 함수, 그리고 당첨등수를 반환해주는 함수들은 유저 단위가 아닌 로또 각각의 단위라고 생각하여 로또객체의 함수로 넣어 구현했습니다.
    객체를 분리하면서 유저가 로또를 생성하고 체크하고 수익률을 계산하는 도메인 로직의 순차적인 실행이 한 눈에 볼 수 있다는 점과 객체 안의 함수를 바꿔도 밖에서 함수를 사용하는 인터페이스는 바뀌지 않아 유동적인 부분이 좋다고 생각했습니다.

- ### **[폴더의 분리]**

  - 저번주차의 가장 아쉬웠던 부분은 폴더와 파일들을 구분하지 않고 한 파일에 구현했다는 점이였습니다. 2주차의 피어리뷰를 공유하면서, 다른 사람들의 폴더 구분 방법과 생각들을 참고하여 이번 과제에서는 constant와 utils로 나눠 구현했습니다.
    특히 utils의 validatior파일을 refactoring하면서 lotto, 당첨번호 생성의 validCheck와 userInput의 validCheck의 중복된 점들을 한번에 관리할 수 있게 되면서 중복된 코드를 줄일 수 있었습니다.

- ### **[도메인 로직에 대한 단위 테스트]**

  - ‘도메인 로직’이란 행위에 대한 의사결정을 담당하고 있는 것이라고 이해했습니다. 예를 들어 ‘로또의 랜덤하게 들어갔는지 체크한다’ 같은 입력을 제공받는 것은 ‘서비스 로직’ 이라고 생각했습니다.
    따라서 로또게임에서의 도메인 로직은 크게 ‘구입금액에 맞게 로또를 생성’, ‘ 로또의 당첨순위 출력’, ‘유저의 수익률, 수익금 계산’ 정도로 나눴고 추가로 validation 체크를 위해 ‘예외 테스트’를 만들어 기능별로 작게 단위테스트를 진행하면서 기능구현을 했습니다.
    로또의 당첨등수(rank)를 테스트하는 부분에서 ‘보너스’ 부분이 들어간 2등, 3등이 바뀌는 오류를 테스트 코드로 바로 알았고, bonus변수에 대입연산자가 아닌 비교연산자로 적는 실수를 바로잡을 수 있었습니다.

- ### **[테스트 코드를 짜는 이유]**

  - 이렇듯 테스트코드는 머릿속으로 구현한 코드가 예상치 못한 문제를 발견할 수 있다는 것이 중요한 이유 중 하나라고 생각합니다. 또한 테스트 코드를 먼저 짜고 개발하려다 보니 객체에서의 함수가 어떤 인자를 받고 어떤 반환 값을 줘야 할지 생각할 수 있어 더욱 안정적이게 코딩을 할 수 있다는 느낌을 받았습니다.
    로또 필드를 없애는 리팩토링 시에도 작은 단위의 테스트 코드가 있으니 수정한 코드의 기능 구현이 동일하게 되었다는 판단을 바로 내릴 수 있어서 시간을 빠르게 단축할 수 있었습니다.

- ### **[비동기 프로그래밍]**

  - 저번 주차 과제를 제출하고 슬랙 커뮤니티를 보던 도중 ‘비동기 프로그래밍’에 대해 간과했던 부분이 있다는 것을 깨달았습니다. 저는 Test Code를 통과하는데 생각이 앞섰을 뿐 사용자가 직접 애플리케이션을 실행하는 것에 대해서는 생각하지 못해서 비동기 프로그래밍을 제대로 제어하지 못했습니다. 이번 로또 게임에서는 직접 실행시켜보며 App.js에서 callBack으로 함수의 기능을 최대한 최소화하면서 비동기를 제어할 수 있게 구현해보려 노력했습니다.

- ### **[값의 하드코딩]**
  - 마지막 리팩토링은 하드 코딩된 값들을 constant로 빼는 것이었습니다. 기본적으로 lotto의 1~45, 6개를 뽑는 것부터, 정규식과 당첨금액들을 constant.js로 빼서 사용하여 리팩토링을 진행했습니다.
    상수를 사용하니 constant에 있는 값들만 바꿔준다면 lotto의 범위, 비용, 당첨금액을 한 번에 바꿀 수 있다는 것을 깨달았고 보기에도 간편할 수 있다는 것을 느꼈습니다.
